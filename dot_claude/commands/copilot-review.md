GitHub Copilotからの現在のPRへのレビューコメントを確認し、対応要否を判断してください。
ユーザーが指定したコメント件数は **$ARGUMENTS 件** です。

## 実行手順

### Step 1: PR情報の取得

現在のブランチに関連するPRを確認：

```bash
gh pr view --json number,title,url,state
```

### Step 2: Copilotからのレビューコメントを取得

PRのレビューコメントを取得し、**Copilot（`copilot[bot]` または `github-advanced-security[bot]`）からのコメントのみ**を抽出してください：

```bash
# PRのレビューコメント（コード行へのコメント）
gh api repos/{owner}/{repo}/pulls/$(gh pr view --json number -q .number)/comments

# PRのレビュー（承認/変更要求時のコメント）
gh api repos/{owner}/{repo}/pulls/$(gh pr view --json number -q .number)/reviews
```

投稿者（`user.login`）が以下のいずれかに該当するコメントのみを対象にしてください：
- `Copilot`
- `copilot[bot]`
- `github-advanced-security[bot]`

### Step 3: 件数確認（重要）

取得したCopilotコメントの件数と、ユーザーが指定した **$ARGUMENTS 件** を比較してください。

- **一致**: そのまま続行
- **不一致**: 警告を表示し、取得漏れがないか確認。ページネーションが必要な場合は追加取得

### Step 4: 各コメントの分析

各レビューコメントについて以下を実行：

1. **コメント内容の確認**: 何を指摘しているか理解
2. **関連コードの確認**: 指摘されたファイル・行を `Read` ツールで確認
3. **対応要否の判断**:
   - **対応必須**: バグ、セキュリティ問題、明らかな改善点
   - **検討**: 設計判断が必要、トレードオフがある
   - **対応不要**: 好みの問題、既に対応済み、誤検知

### Step 5: 実装方針の評価

複数のコメントが以下のパターンに該当する場合、**根本的な設計見直しを提案**してください：
- 同じ種類の問題が繰り返し指摘されている
- アーキテクチャレベルの問題が示唆されている
- テストカバレッジの不足が指摘されている

### Step 6: 結果サマリー

以下の形式で結果を報告：

## 分析結果

**PR**: #番号 - タイトル
**Copilotコメント**: X件（指定: $ARGUMENTS 件）

### 対応必須（X件）
| # | ファイル:行 | 指摘内容 | 対応方針 |
|---|------------|---------|---------|

### 検討（X件）
| # | ファイル:行 | 指摘内容 | 考慮点 |
|---|------------|---------|--------|

### 対応不要（X件）
| # | ファイル:行 | 指摘内容 | 理由 |
|---|------------|---------|------|

### 根本的な見直しが必要か
- [ ] はい → 問題パターンと推奨対応を記載
- [ ] いいえ → 個別対応で十分
